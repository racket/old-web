Enhancements to GUI tester

As things stand, the GUI tester is undocumented, both internally and externally. In the process of reorganizing the code, I have added some comments; more is needed. When I've developed some better structure for the code, I'll describe it here.

Also, there are disparate testing mechanisms for different code bases. For example, MrSpidey has testing mechanisms different from MrEd's. While it is true that the different code bases are now maintained by different individuals, the testers should be usable by others. Eventually, the current maintainers will be elsewhere; also, the person responsible for releases should be able to verify the release.

    4. in the MrEd vocabulary in DrScheme
  • Toggle items in the Language|Configure language|Show Details dialog, and check that they do the right thing
  • Toggle items in the Edit|Preferences dialog, and check behavior
