Parameterization in Zodiac:read

Under certain conditions, DrScheme's Check Syntax button causes an internal error. They are:
  • DrScheme was started with a file,
  • Execute has not been clicked, and
  • There is a symbol in the input.
In those conditions, zodiac:read is called with #f, instead of a parameterization and it fails. Five possible solutions were presented:
  1. Change zodiac:read to accept only a boolean instead of a parameterization. The boolean would be used for case sensitivity, which is the only parameter that the reader currently inspects.
  2. Always initialize the repl when DrScheme is started up.
    • Use threads to avoid some of the cost.
  3. Don't show the Check Syntax button until Execute has been clicked.
  4. Initialize the REPL when Check Syntax is clicked the first time.
  5. Get the case-sensitive from the current parameterization, rather than passing a parameterization.
For version 51, we choose to do solution 1, and for 52 we will do solution 5.
